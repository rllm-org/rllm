# ALFWorld Agent

A multi-turn agent flow for rLLM that trains a model to complete household tasks in the ALFWorld text-based environment via the **AgentFlow protocol**.

## Overview

ALFWorld is a text-based game environment for household tasks. The agent receives text observations describing its surroundings and a list of admissible commands, then outputs a single action per turn. Tasks include picking and placing objects, examining objects under light, heating/cooling/cleaning objects, and multi-object manipulation.

Reward is `1.0` if the agent completes the task, `0.0` otherwise.

## Architecture

```
AgentFlow.run(task, config)
  │
  ├── _init_textworld_env(game_file, max_steps)  # loads .tw-pddl
  └── Multi-turn loop (up to max_steps turns)
        │
        ├── LLM call via OpenAI(base_url=config.base_url)
        │     Model outputs <think>...</think> + ```action\n...\n```
        │
        ├── parse_action() → env.step(action)
        │
        └── Repeat until task won, done, or max_steps
```

## Installation

```bash
uv pip install -e ".[tinker]"                    # rllm + tinker backend
uv pip install --no-deps -e tbmf/alfworld       # this cookbook
```

After installation:

```bash
rllm agent list      # should show "alfworld"
```

## Dataset

The dataset is generated by scanning the `json_2.1.1` game files:

```bash
python tbmf/alfworld/prepare_alfworld_data.py
```

This registers `alfworld/{train,test,eval_ood}` with `DatasetRegistry`.

Prerequisite: ensure the symlink exists:
```
datasets/alfworld/json_2.1.1 → <path to ALFWorld json_2.1.1 data>
```

## Training

### Tinker (single-machine)

```bash
bash tbmf/alfworld/train_tinker.sh
```

### Verl (distributed GPU)

```bash
bash tbmf/alfworld/train_verl.sh
```

### Python API with Hydra overrides

```bash
python tbmf/alfworld/train.py \
    rllm/backend=tinker \
    model.name=Qwen/Qwen3-4B-Instruct-2507 \
    training.group_size=8
```

## Files

| File | Description |
|------|-------------|
| `alfworld_flow.py` | `alfworld_flow` — multi-turn AgentFlow over TextWorld env |
| `alfworld_eval.py` | `alfworld_evaluator` — reward = won? 1 : 0 |
| `prepare_alfworld_data.py` | Scan game files + register train/test/eval_ood splits |
| `train.py` | Python API training script (Hydra config) |
| `train_tinker.sh` | Tinker backend — single-machine training |
| `train_verl.sh` | Verl backend — distributed multi-GPU training |
| `alfworld_pkg/` | ALFWorld TextWorld wrappers and configs |
| `pyproject.toml` | Plugin metadata and entry points |
| `test.py` | Unit tests for action parsing and evaluator |

## Task Types

| ID | Name | Description |
|----|------|-------------|
| 1 | pick_and_place_simple | Pick up an object and place it somewhere |
| 2 | look_at_obj_in_light | Examine an object under a lamp |
| 3 | pick_clean_then_place_in_recep | Clean an object then place it |
| 4 | pick_heat_then_place_in_recep | Heat an object then place it |
| 5 | pick_cool_then_place_in_recep | Cool an object then place it |
| 6 | pick_two_obj_and_place | Pick up two objects and place them |
